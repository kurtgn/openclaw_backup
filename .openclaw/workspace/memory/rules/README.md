# Memory System Overview

- **Hot Memory (`hot.md`)** – Always loaded; confirmed global rules and safeguards.
- **Context Memory (`context.md`)** – Project/domain rules. Organize by project headings.
- **Archive (`archive.md`)** – Retired or inactive patterns. Note archive date + reason.
- **Corrections Log (`corrections.log`)** – Append-only log for every explicit correction, including timestamp, context, summary, and repetition count. When a correction reaches count = 3, ask Misha whether to promote it (and note the decision).
- **Maintenance Routine** – Once per week: deduplicate, keep wording tight, move stale rules to archive, and send Misha a short "what changed" digest.
- **Forgetting Requests** – If asked to forget something, delete the relevant entries from all four files (plus any other storage) and confirm.
