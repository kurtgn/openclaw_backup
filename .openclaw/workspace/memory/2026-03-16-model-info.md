assistant: Да — можно сделать «белый список» через .gitignore, отвергая всё, а затем исключая из игнора нужные расширения. Важно: Git сначала применяет правила .gitignore, а потом работают отрицания (!). Чтобы повторно включить файлы внутри игнорируемых директорий, нужно дать исключение для директорий тоже.

Пример .gitignore, который оставляет только Markdown и JSON (и сам .gitignore) под версионным контролем:

# игнорировать всё
*
# разрешить директории (чтобы отрицания внутри них работали)
!*/
# разрешить .gitignore
!.gitignore
# разрешить MD и JSON в любых папках
!**/*.md
!**/*.markdown
!**/*.json

Короткие пояснения:
- Звёздочка * в начале игнорирует всё (файлы и папки).
- !*/ снимает игнор с директорий, иначе Git не будет смотреть внутрь них и не сможет применить последующие отрицания.
- !**/*.md и !**/*.json включают все .md/.markdown/.json файлы рекурсивно.
- Если у вас уже есть в репо другие отслеживаемые файлы, .gitignore не перестанет их отслеживать автоматически — чтобы убрать их из индекса, используйте git rm --cached <path> или git rm -r --cached . && git add . && git commit (осторожно, сделайте бэкап/проверку перед массовой очисткой).

assistant: Понял — приведу практичный пример записи для ~/.ssh/config и краткие шаги, чтобы подключение «работало» с ключом и с учётом типичной EC2/«amazon» ситуации.

Пример записи в ~/.ssh/config (добавьте в конец файла, замените пути/имена при необходимости):

Host keitaro
    HostName 164.90.180.111
    User root
    Port 22
    IdentityFile ~/.ssh/id_rsa
    IdentitiesOnly yes
    ServerAliveInterval 60
    ForwardAgent yes

Пояснения и рекомендации
- IdentityFile — путь к приватному ключу, которым вы реально будете логиниться. Если ключ имеет другое имя, укажите его.
- IdentitiesOnly yes заставляет клиент использовать только указанный ключ (полезно, если у вас в ssh-agent много ключей).
- ForwardAgent yes — включайте только если доверяете серверу и хотите пробросить агент (например, для дальнейших git‑подключений с сервера).
- ServerAliveInterval 60 — держит соединение живым при простое.
- Убедитесь, что права на приватный ключ строгие: chmod 600 ~/.ssh/id_rsa
- Если ключ защищён паролем, можно добавить его в ssh-agent: eval "$(ssh-agent -s)"; ssh-add ~/.ssh/id_rsa

Если сервер находится за bastion (jump host)
Если нужно проходить через промежуточный хост, используйте ProxyJump:

Host bastion
    HostName bastion.example.com
    User ubuntu
    IdentityFile ~/.ssh/bastion_key

Host keitaro
    HostName 164.90.180.111
    User root
    IdentityFile ~/.ssh/id_rsa
    ProxyJump bastion

Проверка подключения
- Протестировать: ssh -v keitaro  (для подробного вывода добавьте -vvv)
- Если появляется ошибка permission denied — проверьте права ключа и что публичный ключ (~/.ssh/id_rsa.pub) добавлен в ~/.ssh/authorized_keys на сервере.

Если «курсора» вы имеете в виду конкретный клиент (VS Code / Cursor / другой)
- Для VS Code Remote SSH: в настройках расширения укажите тот же Host alias (keitaro) — VS Code использует ~/.ssh/config автоматически.
- Для других GUI/инструментов обычно можно либо импортировать  ~/.ssh/config, либо прямо указать те же параметры.
